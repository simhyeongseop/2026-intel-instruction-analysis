- **필수**: 대문자로 시작

**패턴 예시**:
```
유효한 명령어: ADD, MOV, VFMADD132PD, CMOV
무효한 명령어: aad, 1ADD, -MOV
```

## 📌 다음 단계 (3회 미팅까지의 과제)

### 과제 1️⃣: Opcode와 Instruction 분리

**목표**: 명령어에 섞여있는 Opcode와 Instruction을 분리

**주의사항**: 특히 **VEX/EVEX/REX 접두사** 처리 필수
- VEX를 V로 시작하는 명령어와 구분
- 벡터 연산 명령어(VMOVUPS 등) 정확한 분리

**예시**:
```
입력: EVEX.128.0F.W0 10 /rVMOVUPS xmm1 {k1}{z}
출력 Opcode: EVEX.128.0F.W0 10 /r
출력 Instruction: VMOVUPS xmm1 {k1}{z}
```

### 과제 2️⃣: 오퍼랜드 분리

슬래시(/)로 묶여있는 오퍼랜드 분해:
```
r/m16 → r16, m16 (분리)
m16&16 → m16 (정리)
xmm2/m128 → xmm2, m128
```

### 과제 3️⃣: 오퍼랜드 그룹화

- 추출한 오퍼랜드를 중복 없는 집합(Set)으로 정리
- 의미별 분류:
  - 레지스터 크기별 (8bit, 16bit, 32bit, 64bit)
  - 메모리 접근 크기별
  - 세그먼트 레지스터
  - 즉시값 (imm8, imm16, etc.)
