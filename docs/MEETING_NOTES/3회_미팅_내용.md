## 📌 미팅 주제

Opcode 분리, Operand 분리 및 분류 최종 완성

---

## 📊 2단계 진행 결과

### ✅ Opcode와 Instruction 분리 완료

**방식**:
- EVEX/VEX/REX 접두사 정규표현식 기반 추출
- `/r`, `/m` 등의 인코딩 표시 기준으로 분할

**예시**:
```
입력: EVEX.128.0F.W0 10 /rVMOVUPS xmm1 {k1}{z}
Opcode: EVEX.128.0F.W0 10 /r
Instruction: VMOVUPS xmm1 {k1}{z}
```

**통계**:
- Opcode 포함 명령어: 2,085개
- Opcode 없음 (일반 명령어): 3,043개
- 총: 5,128개

---

## 📊 3단계 진행 결과

### ✅ 오퍼랜드 분리 (1로 확장)

**목표**: r/m16 같은 combined operand를 하나의 명령어 = 여러 행으로 확장

**처리 방식**:
```
ADC r/m16  →  [ADC r16], [ADC m16]  (2줄)
BOUND m16&16  →  [BOUND m16]  (1줄)
VMOVUPS xmm2/m128  →  [VMOVUPS xmm2], [VMOVUPS m128]  (2줄)
```

**통계**:
- 원본: 5,128개 명령어
- 확장 후: 6,080개 (952개 추가 생성)

### ✅ 오퍼랜드 추출

**방식**:
- 정규표현식을 사용한 패턴 기반 추출
- 레지스터, 메모리, 즉시값, 주소 지정형 등을 구분

**추출된 오퍼랜드 분류**:
- 일반 레지스터: r8, r16, r32, r64, al, ax, eax, rax
- 벡터 레지스터: xmm0-31, ymm0-31, zmm0-31
- 메모리 클래스: m8, m16, m32, m64, m128, m256, m512
- 즉시값: imm8, imm16, imm32, imm64
- 주소 지정: rel8, rel16, rel32, ptr16:16, ptr16:32
- 마스크: k0-k7

**통계**: 229개 고유 오퍼랜드

### ✅ 오퍼랜드 분류

**15개 카테고리로 분류**:
